If we want to change the name of a block (ie. change `forever_loop` to loop_forever):
1. Change the block name in the block-types file (ie. logic-block-types.ts)
2. Change all occurrences throughout the common-ts repo. Publish.

Client/server: `npm run updateCommon`
3. Update the front-end blockly `*-blocks.ts` files accordingly.

The DB migration, and pushing client up should happen at the same time, or we risk
	-the DB saving old data (if backend updated after client),
	-or the client retrieving old data (if the DB is updated before client)
4. Create a migration in the DB, for both the sandbox_project, and the challenge_sandbox
	ie: UPDATE sandbox_project 
		SET sandbox_json = replace(replace(replace(replace(
		sandbox_json::text, 
		'esp32_led_control', 'control_all_leds'), 
		'controls_repeat_ext', 'for_loop'), 
		'esp32_delay', 'delay'), 
		'esp32_loop', 'forever_loop')::jsonb
		WHERE sandbox_json::text ~ '(esp32_led_control|controls_repeat_ext|esp32_delay|esp32_loop)';

	- This migration should be done on the prisma level, to keep a historical record
